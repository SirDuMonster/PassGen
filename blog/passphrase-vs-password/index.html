Passphrase vs Password: Which is More Secure?

Understanding when to use random characters vs memorable words

The famous XKCD comic suggested that "correct horse battery staple" is both more secure and more memorable than "Tr0ub4dor&3". But is this actually true? Let's examine the science behind passphrases vs passwords.

What's the Difference?

Traditional Password

Kj9#mP2x$Lw4

Random mix of characters, numbers, and symbols. Hard to remember, hard to crack.

Passphrase

correct-horse-battery-staple

Multiple random words. Easier to remember, can be equally secure with enough words.

The Math Behind Security

Password strength is measured in entropy (bits of randomness). Higher entropy = more combinations = harder to crack.

Password Entropy

A 12-character password using all character types (95 possible characters):

Entropy = 12 × log₂(95) = 78.8 bits

Passphrase Entropy

A 4-word passphrase from a 7,776-word dictionary (like Diceware):

Entropy = 4 × log₂(7776) = 51.7 bits

Wait—the password has more entropy! So why do people recommend passphrases?

The Memorability Factor

Security that you can't use is no security at all. If a password is too complex:

  • You'll write it down insecurely
  • You'll reuse simpler passwords
  • You'll reset it constantly
  • You'll use keyboard patterns that feel random but aren't

A 6-word passphrase has 77.5 bits of entropy—similar to our complex password—but most people can memorize it after a few uses.

Security Comparison

Factor Password Passphrase
Entropy per character ~6.5 bits ~12.9 bits/word
Memorability Poor Good
Typing speed Slow (special chars) Fast (normal words)
Works everywhere Yes Usually (length limits)
Resistant to dictionary attacks Yes If random words
Mobile entry Painful Easy

When to Use Each

Use a Passphrase For:

  • Master passwords: Password manager, disk encryption
  • Frequently typed: Computer login, WiFi password
  • Must memorize: Any password you can't store in a manager
  • Shared with others: Family WiFi, shared accounts

Use a Random Password For:

  • Stored in password manager: You don't need to remember it
  • Length-restricted sites: Some limit to 16-20 characters
  • Maximum security: When you want the highest possible entropy
  • API keys & tokens: Never manually typed

Creating Strong Passphrases

Do:

  • Use at least 4-6 random words
  • Use a word list (Diceware, EFF word list)
  • Add a number or symbol if required
  • Use spaces or separators between words

Don't:

  • Use song lyrics, quotes, or book titles
  • Choose words yourself (humans are bad at random)
  • Use common phrases ("iloveyou", "letmein")
  • Use personal information (pet names, birthdays)

Good Passphrases

timber-lunar-harvest-ocean-crystal

Elephant$Piano7Clouds!River

correct horse battery staple 42

Bad Passphrases

ilovemydog (common phrase)

ToBeOrNotToBe (famous quote)

johnsmith1990 (personal info)

The Best of Both Worlds

For maximum security and memorability, combine approaches:

  1. Generate a random passphrase (4-6 words)
  2. Add a number somewhere
  3. Capitalize one word unusually
  4. Use a unique separator

timber.LUNAR.harvest.ocean.42

This passphrase has the memorability of words plus added entropy from variations.

Verdict: Use Both

The passphrase vs password debate misses the point. Use the right tool for the situation:

  • Passphrases for things you need to remember and type
  • Random passwords for everything stored in your password manager

A password manager with a strong passphrase master password, containing random passwords for each account, gives you the best of both worlds.
