How Password Crackers Work: Understanding the Threat

Know your enemy to better defend yourself

Understanding how attackers crack passwords helps you create better ones. This guide explains the main techniques hackers use—and how to defend against each.

How Passwords Are Stored

First, understand that responsible websites don't store your actual password. They store a hash—a one-way mathematical transformation.

Your password: MySecret123

SHA-256 hash: 7f2ababa62...(64 characters)

When you log in, the site hashes your input and compares it to the stored hash. If they match, you're authenticated—without ever storing your actual password.

Password cracking is the process of finding what input produces a specific hash.

Cracking Method #1: Brute Force

The simplest approach: try every possible combination.

How It Works

Start with "a", then "b", "c"... then "aa", "ab", "ac"... and continue until you find a match.

Speed

Modern GPUs can try billions of hashes per second. A single RTX 4090 can test:

  • MD5: ~165 billion/second
  • SHA-256: ~23 billion/second
  • bcrypt: ~184,000/second (designed to be slow)

Defense

Use long passwords. Each additional character multiplies crack time exponentially:

  • 8 characters: Hours to days
  • 12 characters: Months to years
  • 16 characters: Millions of years

Cracking Method #2: Dictionary Attacks

Instead of random combinations, try common words and passwords.

How It Works

Use lists of:

  • Dictionary words in multiple languages
  • Previously leaked passwords
  • Common names and phrases
  • Keyboard patterns (qwerty, 123456)

The "RockYou" List

In 2009, the RockYou breach exposed 32 million plaintext passwords. This list revealed that most people use predictable passwords. The top 10:

  1. 123456
  2. 12345
  3. 123456789
  4. password
  5. iloveyou
  6. princess
  7. 1234567
  8. rockyou
  9. 12345678
  10. abc123

Defense

Never use dictionary words, common phrases, or passwords from breach lists. Use randomly generated passwords.

Cracking Method #3: Rule-Based Attacks

Apply transformations to dictionary words to catch "clever" variations.

Common Rules

  • Capitalization: password → Password, PASSWORD
  • Append numbers: password → password1, password123, password2024
  • Append symbols: password → password!, password@
  • Leet speak: password → p@ssw0rd, p455w0rd
  • Combinations: password → P@ssw0rd123!

Defense

These rules are well-known. "P@ssw0rd!" isn't clever—it's one of the first variations tested. True randomness defeats rule-based attacks.

Cracking Method #4: Rainbow Tables

Pre-compute hashes for millions of passwords, then just look them up.

How It Works

Instead of computing hashes during the attack, compute them in advance and store in a massive lookup table. When you get a hash, just search the table.

Limitation

Tables can be enormous (terabytes) and only work for specific hash types. They're defeated by salting—adding random data to each password before hashing.

Defense

This is the website's responsibility. Proper password hashing (bcrypt, Argon2) includes salting automatically. As a user, use unique passwords so a cracked hash on one site doesn't work elsewhere.

Cracking Method #5: Targeted Attacks

Research the victim to build custom wordlists.

Information Sources

  • Social media profiles
  • Public records (addresses, family names)
  • Previous data breaches
  • Company websites (for work accounts)
  • Physical surveillance (pet names, car models)

How It Works

If an attacker knows your dog is named Max, they try: Max, Max123, ILoveMax, Max2020, M@x!, etc. Combined with rule-based attacks, personal information dramatically reduces the search space.

Defense

Never use personal information in passwords. Random passwords can't be researched.

Cracking Method #6: Credential Stuffing

Not cracking at all—just reusing stolen credentials.

How It Works

When Site A gets breached, attackers try those username/password combinations on Sites B, C, D, etc. If you reuse passwords, one breach compromises all your accounts.

Scale

Billions of credentials are available from past breaches. Automated tools can test thousands of sites in hours.

Defense

Use unique passwords for every account. A password manager makes this practical.

Cracking Method #7: Phishing

Why crack the password when you can just ask for it?

How It Works

Create a fake login page that looks identical to the real one. Send victims links via email, SMS, or ads. When they "log in," capture their credentials.

Defense

  • Check URLs carefully before entering credentials
  • Use 2FA (phished password alone isn't enough)
  • Use a password manager (won't autofill on fake sites)
  • Be suspicious of unexpected login requests

How Long Would Your Password Take to Crack?

Password Type Example Crack Time*
Common password password123 Instant
Dictionary + rules P@ssw0rd! Seconds
8 random chars Kj9#mP2x Days
12 random chars Kj9#mP2x$Lw4 Centuries
16 random chars Kj9#mP2x$Lw4nQ8r Billions of years

*Assuming offline attack against weak hash with modern GPU cluster

Summary: How to Stay Safe

  1. Use random passwords—defeats dictionary and targeted attacks
  2. Use long passwords (16+)—defeats brute force
  3. Use unique passwords—defeats credential stuffing
  4. Enable 2FA—defeats phishing (partially)
  5. Use a password manager—makes all of the above practical
