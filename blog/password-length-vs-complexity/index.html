Password Length vs Complexity: What Actually Matters

The mathematical truth behind password strength

You've been told to use uppercase, lowercase, numbers, and symbols. But is "Kj9#" really better than "correcthorse"? Let's look at the math.

The Formula for Password Strength

Password strength is measured in entropy, calculated as:

Entropy = Length × log₂(Pool Size)

Where:

  • Length = number of characters
  • Pool Size = possible characters per position

Character Pool Sizes

Characters Used Pool Size Bits per Character
Lowercase only (a-z) 26 4.7 bits
+ Uppercase (A-Z) 52 5.7 bits
+ Numbers (0-9) 62 5.95 bits
+ Symbols 95 6.57 bits

Length vs Complexity: Real Examples

Password A: Complex but Short

Kj9#mP2$

  • Length: 8 characters
  • Pool: 95 (all types)
  • Entropy: 8 × 6.57 = 52.6 bits

Password B: Simple but Long

thequickbrownfox

  • Length: 16 characters
  • Pool: 26 (lowercase only)
  • Entropy: 16 × 4.7 = 75.2 bits

Winner: The longer, simpler password!

Password B has 43% more entropy despite using only lowercase letters.

Why Length Wins Mathematically

Adding complexity multiplies possibilities linearly. Adding length multiplies possibilities exponentially.

Consider 8 vs 16 characters with all character types (95):

  • 8 characters: 95⁸ = 6.6 × 10¹⁵ combinations
  • 16 characters: 95¹⁶ = 4.4 × 10³¹ combinations

Doubling length doesn't double strength—it squares it (and then some).

But There's a Catch

This math assumes random selection. In practice:

The Dictionary Attack Problem

"thequickbrownfox" isn't 16 random lowercase letters—it's a famous phrase. Attackers try common phrases first, making this password much weaker than the math suggests.

The Pattern Problem

"Password123!" has theoretical entropy of ~73 bits, but it's one of the most common passwords because humans follow predictable patterns.

The Real Answer: Both (With Randomness)

The best passwords are:

  1. Long (16+ characters)
  2. Complex (all character types)
  3. Random (no patterns or words)

Weak Despite Complexity

P@ssw0rd!

Follows common patterns, easily guessed

Weak Despite Length

iloveyousomuch2024

Common phrase + year, in dictionaries

Strong: Long + Complex + Random

Kj9#mP2x$Lw4nQ8r

~105 bits of entropy

Practical Recommendations

For Passwords You Must Memorize

Use a random passphrase: 5-6 random words provides good entropy with better memorability.

timber-ocean-crystal-harvest-moon

This gives ~64 bits of entropy and is much easier to remember than random characters.

For Passwords Stored in a Manager

Use maximum randomness: 16-20 characters using all character types. You don't need to remember it, so optimize for security.

Kj9#mP2x$Lw4nQ8r!Yz

Minimum Thresholds

Account Type Minimum Length Minimum Entropy
Low-value accounts 12 characters ~60 bits
Important accounts 16 characters ~80 bits
Critical (master password, banking) 20+ characters ~100+ bits

What About Password Requirements?

Many sites still require "at least one uppercase, one number, one symbol." This is outdated guidance that:

  • Creates predictable patterns (Capital at start, number at end)
  • Doesn't ensure randomness
  • Often sets maximum lengths that hurt security

NIST (National Institute of Standards and Technology) now recommends prioritizing length over complexity rules and allowing up to 64 characters.

The Bottom Line

If you had to choose one, length beats complexity. But you don't have to choose—use a password manager and generate long, complex, random passwords for everything.

The formula is simple:

Strong Password = Length + Complexity + Randomness
