Password Security Glossary

Essential terms and definitions for understanding password security

B C D E H K M P R S T

B

Brute Force Attack

A method of cracking passwords by systematically trying every possible combination of characters until the correct one is found. Brute force attacks become exponentially harder as password length and character variety increase.

Example: An 8-character lowercase password has 208 billion possible combinations. Adding uppercase, numbers, and symbols increases this to 6 quadrillion.

C

Credential Stuffing

An attack where stolen username/password pairs from one data breach are automatically tried against other websites. This exploits the common practice of password reuse across multiple accounts.

Prevention: Use unique passwords for every account.

Cryptographic Hash Function

A mathematical algorithm that converts any input into a fixed-size string of characters. Good password systems store hashes, not actual passwords. Common algorithms include bcrypt, Argon2, and PBKDF2.

D

Dictionary Attack

A password cracking technique that uses a list of common words, phrases, and known passwords to guess credentials. More efficient than brute force for passwords based on real words.

Defense: Avoid dictionary words; use random characters or multiple unrelated words.

Data Breach

An incident where unauthorized individuals gain access to sensitive data, often including passwords or password hashes. Major breaches have exposed billions of credentials.

E

Entropy

A measure of password randomness, expressed in bits. Higher entropy means more possible combinations and greater resistance to guessing. Calculated as: Length √ó log‚ÇÇ(Character Pool Size).

Example: A 16-character password using all character types (95 characters) has ~105 bits of entropy‚Äîconsidered very strong.

H

Hashing

The process of converting a password into a fixed-length string using a hash function. Unlike encryption, hashing is one-way‚Äîyou can't reverse it to get the original password. Websites should store password hashes, never plain-text passwords.

K

Keylogger

Malicious software that records keystrokes to capture passwords as they're typed. Can be software-based or a physical device attached to a keyboard.

Defense: Use anti-malware software, 2FA, and password managers with auto-fill.

M

Multi-Factor Authentication (MFA)

Security requiring two or more verification methods: something you know (password), something you have (phone), or something you are (fingerprint). See also: Two-Factor Authentication.

Master Password

The single password used to unlock a password manager vault. Should be strong and memorable since it protects all other passwords. Consider using a passphrase.

P

Passphrase

A password consisting of multiple random words, often separated by spaces or symbols. Easier to remember than random characters while maintaining high entropy.

Example: "correct-horse-battery-staple" (from XKCD) has ~44 bits of entropy with 4 words.

Password Manager

Software that securely stores, generates, and auto-fills passwords. Allows using unique, strong passwords for every account while only remembering one master password.

Phishing

A social engineering attack that tricks users into revealing passwords by posing as a legitimate website or service. Often delivered via email or fake login pages.

Defense: Verify URLs, use 2FA, never click suspicious links.

R

Rainbow Table

A precomputed table of password hashes used to crack passwords quickly. Instead of computing each hash, attackers look up the hash in the table. Defeated by salting.

S

Salt

Random data added to a password before hashing. Ensures that identical passwords produce different hashes, defeating rainbow table attacks. Each password should have a unique salt.

Social Engineering

Psychological manipulation to trick people into revealing sensitive information. Includes phishing, pretexting, and other non-technical attacks.

T

Two-Factor Authentication (2FA)

Security method requiring two different types of verification. Typically a password plus a code from an authenticator app, SMS, or hardware key. Significantly reduces account compromise risk.

Best practice: Enable 2FA on all accounts that support it, especially email and financial accounts.

TOTP (Time-Based One-Time Password)

A type of 2FA that generates temporary codes (usually 6 digits) that change every 30 seconds. Used by authenticator apps like Google Authenticator and Authy.
